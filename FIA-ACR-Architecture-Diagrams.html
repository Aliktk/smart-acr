Role-based interaction flows

// Who does what · workflow chain · access boundaries per role

Clerk

Initiates ACR packets, fills reporting period dates, submits to Reporting Officer. Can correct and resubmit when returned.

✓ Create / fill / submit ACR

✓ Own ACRs + queue view

✓ Upload signature/stamp

✓ Correct on return

✗ Edit RO/CSO sections

✗ Access other clerks' ACRs

✗ User management / audit logs

Reporting officer

Receives ACR, fills assessment, forwards to CSO or (APS only) directly to Secret Branch. Can return with remarks.

✓ View assigned ACRs

✓ Fill RO section + upload

✓ Forward to CSO or SB

✓ Return to clerk

✗ Edit clerk section

✗ Act on another RO's ACRs

✗ Archive / audit access

Countersigning officer

Countersigns where required — not APS_STENOTYPIST. Reviews prior sections (read-only), fills CSO section, submits to SB.

✓ View all sections (read)

✓ Fill CSO section + upload

✓ Submit to Secret Branch

✓ Return to RO or clerk

✗ APS_STENOTYPIST ACRs

✗ Edit clerk / RO sections

✗ Act on another CSO's ACRs

Secret branch

Two-stage process: desk assignment + review completion, then final verification and archive. Can return to any prior stage.

✓ Assign desk (AD/DA1–DA4)

✓ Complete SB review stage

✓ Verify and create archive

✓ Return to clerk/RO/CSO

✗ Edit any officer section

✗ User provisioning

Super admin / IT ops

Full platform management. Provisions accounts, assigns roles and org scope, manages hierarchy. Read-only on all ACRs.

✓ Create/deactivate users

✓ Assign roles + org scope

✓ Read all ACRs (no edit)

✓ Full audit log access

✗ Workflow transitions

✗ IT_OPS cannot modify SUPER_ADMIN

DG / Executive viewer

Read-only oversight. DG + EXEC_VIEWER see all records. WING_OVERSIGHT and ZONAL_OVERSIGHT are scope-limited.

✓ DG: all ACRs + analytics

✓ WING_OVERSIGHT: wing-scoped

✓ ZONAL_OVERSIGHT: zone-scoped

✓ Dashboard + archive browse

✗ No form editing

✗ No workflow transitions
